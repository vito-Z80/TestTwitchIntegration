1) Создать папку в текущей директории с именем будущего аватара.
2) Создать папки состояния аватара внутри папки аватара. Может быть до 4-х папок состояний: idle, left, right, attack.
    Если есть right но нет left, то left добавится уже внутри приложения. и наоборот.
    Если есть attack, то аватар может реагировать на команду атаки.
    Если есть idle то аватар может перестать движение на небольшой промежуток времени применяя анимацию idle.
    Если idle нет то персонаж аватар всегда в движении.
    
Призвать аватара:
    _i_avatarName
        _i_toad
        _i_scull
        
    avatarName - это имя папки аватара в любом символьном регистре.
    Эта команда так-же сменит текущего аватара на нового, если имя нового аватара есть в папке Avatars
    
Атаковать:
    _attack @userName
    
    Команда будет выполнена если у аватара есть состояние attack среди папок аватара с нужными спрайтами атаки.
    
    
Спрайты аватаров должны быть из старых 8-16-бит игр оригинальных размеров, иначе они будут огромными на экране.
